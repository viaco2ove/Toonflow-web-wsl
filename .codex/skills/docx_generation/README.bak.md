# 技能描述
## 主要功能
一个通过md 文件生成docx 文件的技能。
默认功能是一个或多个md文件 生成一个docx文件 

## 额外功能：
1.docx文件  转换为md 文件和排版文件和图片资源
2.docx文件  转换为包含格式（可以是html嵌入到md 文件）的md 文件 加图片资源
3.通过docx文件 分析出排版文件
4.一个或多个md文件通过排版文件 生成一个docx文件

## 可行性分析
 可行性结论（先给结论）                                                                                                                                                                                                    
                                                                                                                                                                                                                            
  - docx → md（含格式/HTML）+ 图片资源：可行，常用方案是 pandoc 或 mammoth 做 docx→HTML，再将 HTML 直接嵌入 md。                                                                                                            
  - docx → md（纯 Markdown）：部分可行，会损失复杂排版（浮动图、文本框、分栏、复杂表格等）。                                                                                                                                
  - “排版文件”作为中间/控制层：可行，但需要明确格式与映射规则，否则无法稳定还原。                                                                                                                                           
                                                                                                                                                                                                                            
  docx → md（含 HTML + 图片资源）可行性                                                                                                                                                                                     
                                                                                                                                                                                                                            
  - docx 本质是 zip，图片位于 word/media，提取图片资源没有问题。                                                                                                                                                            
  - docx 结构化信息（标题、段落、表格、列表、脚注）可以解析并转为 HTML。                                                                                                                                                    
  - Markdown 支持原生 HTML，可把复杂结构保留为 HTML 块（如表格、图文混排、复杂样式）。                                                                                                                                      
                                                                                                                                                                                                                            
  主要限制（必须说明）                                                                                                                                                                                                      
                                                                                                                                                                                                                            
  - 不可能完全无损：浮动对象、复杂表格样式、文本框、分栏、分页控制在 md 中要么丢失、要么退化为 HTML。                                                                                                                       
  - 你需要接受“md 用于内容 + HTML 用于结构”的折中。   

  “排版文件”是否可行？什么格式合适？                                                                                                                                                                                        
  **可行。**建议把“排版文件”定义为“样式与结构映射”的配置文件或模板。常见且可落地的选型：                                                                                                                                    
                                                                                                                                                                                                                            
  方案 A（推荐）：YAML/JSON 样式映射文件                                                                                                                                                                                    
                                                                                                                                                                                                                            
  - 形式：layout.yaml / layout.json                                                                                                                                                                                         
  - 内容建议：                                                                                                                                                                                                              
      - docx 样式名 → md/HTML 标签或类名映射                                                                                                                                                                                
      - 段落/标题层级规则（如 Heading 1 → <h1>）                                                                                                                                                                            
      - 列表编号/项目符号映射                                                                                                                                                                                               
      - 表格样式映射（是否用 HTML 表格）                                                                                                                                                                                    
      - 图片默认宽高、对齐方式                                                                                                                                                                                              
      - 页眉/页脚处理规则（是否写入 md 前言）                                                                                                                                                                               
  - 适配性强，便于版本控制                                                                                                                                                                                                  
                                                                                                                                                                                                                            
  方案 B：docx 模板（.dotx / .docx 作为“排版文件”）                                                                                                                                                                         
                                                                                                                                                                                                                            
  - 用于“md → docx”的反向生成                                                                                                                                                                                               
  - 优点：能直接用 Word 控制字体、行距、分页                                                                                                                                                                                
  - 缺点：不适合“docx → md”的解析说明，只能当“输出模板”                                                                                                                                                                     
                                                                                                                                                                                                                            
  方案 C：HTML 模板 + CSS（用于预览或二次导出）                                                                                                                                                                             
                                                                                                                                                                                                                            
  - 把 md 中的 HTML 渲染成完整网页                                                                                                                                                                                          
  - CSS 负责“排版风格”，但 CSS 不等于 docx 样式语义                                                                                                                                                                         
  - 适合“md → HTML → PDF”                                                                                                                                                                                                   
                                                                                                                                                                                                                            
  总结：排版文件最实用的是 “样式映射配置（YAML/JSON）+ 参考 docx 模板” 的组合。                                                                                                                                             
                                                                                                                                                                                                                            
# 功能对应的实现：                                                                                                                                                                                
                                                                                                                                                                                                                            
  - docx → md + 图片：pandoc 或 mammoth                                                                                                                                                                                     
  - docx → md（保留复杂格式）：先转 HTML，再嵌入 md
  - 版式/样式控制：layout.yaml + reference.docx 的双层策略
  - 解析复杂段落与样式：需要明确 docx 里使用的样式名（例如标题、正文、引用、警示块）
