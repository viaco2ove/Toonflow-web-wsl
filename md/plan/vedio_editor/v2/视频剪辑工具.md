# 视频剪辑工具 v2 改进方案

## 0. 目标
把 `v1` 里“可用但不稳定”的时间轴编辑，升级为“功能闭环 + 行为一致 + 可维护”的版本。  
重点是补齐未完成功能，并收敛当前实现里的结构性风险。

## 1. 现状问题清单（对照 v1）

### P0（必须先做，不做会持续返工）

1. 未完成功能仍缺失
- `分镜图重生成`：时间轴编辑器里没有入口，也没有和分镜模块形成回写联动。
- `多音频切换`：当前仅支持 `audioPath/ttsAudioPath` 单路选择，缺少同片段多音频版本管理与切换。

2. 时间轴核心交互仍不够稳
- 时间轴 `mousedown` 绑定在整个 `timelineScroller`，容易与轨道按钮/片段点击产生冲突触发。
- 排序相关函数存在但未形成可见交互闭环（例如 `moveClip` 未被真实交互调用）。

3. 全屏与滚动体验在不同窗口下不一致
- 全屏、放大、预览全屏是三套行为，入口分散，用户心智负担高。
- 滚动区责任边界不清晰，存在“某些内容跟随滚动、某些不跟随”的历史问题。

### P1（影响体验质量）

1. 预览同步模型不可靠
- 视频、音频、台词使用独立媒体对象播放，长时间播放会出现漂移风险。
- 轨道静音/可视切换时，预览重同步策略是“重设 source + currentTime”，易产生顿挫。

2. 轨道语义混淆
- 台词生成时同时写入 `audioPath` 和 `ttsAudioPath`，导致“音频轨/台词轨”边界不清晰。
- 轨道是“多轨道 UI”，但不是“多版本音频资产模型”，后续做多音频切换会很痛苦。

3. 任务与错误反馈不完整
- 依赖大量散落 `axios.post`，失败信息和重试策略不统一。
- 异步任务（台词生成、上传音频、排序保存）缺少一致的状态管理与取消机制。

### P2（工程质量）

1. 组件过大
- `videoTimelineEditor.vue` 约 `1333` 行，状态、视图、接口、副作用全耦合在一个文件里。

2. 代码可维护性不足
- 接口调用路径分散，重复字段映射和重复刷新逻辑较多。
- 局部函数职责重叠，存在未使用逻辑，长期会继续堆积技术债。

3. 缺少可回归验证
- 目前仅靠手工点测，没有对关键交互（seek、mute、轨道切换、全屏）建立稳定回归脚本。

## 2. v2 改进方案

## 2.1 功能闭环（先补齐）

1. 分镜图重生成
- 在时间轴选中片段后，新增“重生成分镜”入口。
- 复用现有分镜模块接口能力，回写当前片段关联分镜并触发局部刷新。
- 验收：单片段可重生分镜，不影响其他片段，回写后时间轴封面与详情一致。

2. 多音频切换
- 数据模型升级：每个片段支持 `audioAssets[]`（原音、TTS、上传音频、历史版本）。
- UI 增加“音频版本选择器 + A/B 试听 + 一键设为主音轨”。
- 验收：同片段至少 3 个音频版本可切换，切换后播放与导出使用同一主版本。

## 2.2 交互一致性（再做体验）

1. 全屏统一策略
- 统一成两种模式：`编辑器全屏`、`预览全屏`，并明确优先级与互斥关系。
- 全屏按钮固定在可见位置，避免受标题布局影响。

2. 时间轴输入分层
- 将 `seek` 热区限定到标尺/片段区域，按钮区和表单区不响应拖拽 seek。
- 拖拽、点击、滚动事件拆分，避免互相抢事件。

3. 滚动职责单一化
- 一个纵向滚动容器负责全局上下浏览。
- 时间轴内部只负责横向缩放与横向滚动。

## 2.3 结构重构（并行推进）

1. 拆分组件
- `TimelineHeader.vue`
- `TimelinePreview.vue`
- `TimelineTracks.vue`
- `TimelineInspector.vue`

2. 拆分逻辑
- `useTimelineModel.ts`（片段、轨道、标尺、playhead）
- `useTimelinePlayback.ts`（播放、seek、同步）
- `useTimelineActions.ts`（保存、上传、生成、刷新）

3. 服务层统一
- `src/services/videoConfigService.ts`
- `src/services/voiceService.ts`
- `src/services/storyboardService.ts`
- 统一错误处理、重试、loading key、取消请求。

## 3. 实施顺序

### 里程碑 M1（功能止血）
- 完成 `分镜图重生成` 与 `多音频切换` 的最小闭环。
- 修正时间轴事件冲突，保证按钮点击不触发误 seek。

### 里程碑 M2（体验稳定）
- 完成全屏与滚动一致性改造。
- 统一预览同步策略，降低音视频漂移与切换抖动。

### 里程碑 M3（结构治理）
- 组件/逻辑/服务分层落地。
- 清理未使用代码，减少单文件复杂度。

## 4. 验收标准（必须可量化）

1. 功能
- 可对任意片段执行“分镜图重生成”并回写成功。
- 任意片段支持多音频版本切换，切换后播放结果与导出一致。

2. 交互
- 全屏按钮始终可见且可用。
- 时间轴按钮点击误触发 seek 的复现率为 0。
- 纵向滚动只存在 1 个主滚动容器，不再出现越界内容。

3. 工程
- 时间轴主组件代码量降到可控范围（目标：拆分后单文件不超过 400 行）。
- 关键流程有最小回归清单：打开编辑器、seek、播放、切轨、静音、上传、生成、保存。

## 5. 备注
- v2 不追求一次性“大重写”，采用“先闭环功能，再统一交互，再拆结构”的顺序，避免再次出现“代码很多但需求没闭环”的情况。
